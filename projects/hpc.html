High-Performance 2D Heat Diffusion Solver

HPC Parallelization March 2026 MPI OpenMP C Valgrind

Overview

Part of the High-Performance Computing (HPC) course, this project focuses on solving the 2D heat equation using a five-point stencil computation. The goal was to implement an efficient parallel solver and analyze its scaling behavior on real HPC clusters.

Implementation

  • Hybrid parallelism: MPI is used for inter-node communication (distributing rows of the 2D grid across processes), while OpenMP parallelizes the inner loops within each node across threads.
  • Halo exchanges: At each time step, neighboring MPI processes exchange boundary rows (ghost cells) to ensure correct stencil computation at process boundaries.
  • Periodic boundary conditions: Optional support for periodic boundaries, wrapping the grid edges.
  • Domain decomposition: The 2D grid is decomposed along one axis across MPI ranks, balancing load across processes.

Profiling and analysis

Strong and weak scaling experiments were conducted on the Leonardo and Orfeo HPC clusters. Shell scripts automate multi-node job submission; Python notebooks post-process timing data and produce scaling plots. Valgrind cache profiling was used to identify memory bottlenecks and optimize data access patterns.

Key findings

The hybrid approach achieves near-linear strong scaling up to a moderate number of processes, after which communication overhead begins to dominate. Weak scaling remained efficient across the tested configurations, confirming that the domain decomposition strategy effectively balances computation and communication.
